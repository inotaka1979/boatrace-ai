## 0. 設計原則

### 0.1 4 つの基本原則
1. **絶対値より相対値**: 場別・選手別の正規化を最優先（場差 / 個人差を吸収）
2. **的中率より期待値**: EV (Expected Value) を判断軸の中心に置く
3. **層分離**: データ取得 / 特徴量計算 / 予測 / 戦略を疎結合に保つ
4. **検証可能性**: 全変更をバックテストで定量評価できる構造にする

#### R-03 / R-11: EV ベース買い目選定
**ファイル**: `index.html` 内の新ヘルパ
```js
/**
 * EV ベースで買い目を選定。
 * @param {Object<string,number>} probs  例 {"1-2-3": 0.18, "1-3-2": 0.12, ...}
 * @param {Object<string,number>} odds   オッズ（同じキー）
 * @param {Object} opt
 *   - evMin: EV 閾値（既定 1.15）
 *   - maxBets: 最大点数（既定 8）
 *   - kellyFrac: Kelly 比率（half-Kelly = 0.5、既定）
 *   - bankroll: 資金（円）
 * @returns {Array<{combo, ev, prob, odds, stakeRatio, stakeYen}>}
 */
function selectBetsByEV(probs, odds, opt={}){
  const {evMin=1.15, maxBets=8, kellyFrac=0.5, bankroll=10000} = opt;
  const ranked = Object.keys(probs)
    .filter(k => odds[k] && probs[k])
    .map(k => ({
      combo: k,
      prob: probs[k],
      odds: odds[k],
      ev: probs[k] * odds[k],
    }))
    .filter(b => b.ev >= evMin)
    .sort((a, b) => b.ev - a.ev)
    .slice(0, maxBets);
  // Kelly: f* = (b·p - q) / b, ただし b = odds-1, q = 1-p
  ranked.forEach(b => {
    const f = ((b.odds - 1) * b.prob - (1 - b.prob)) / (b.odds - 1);
    b.stakeRatio = Math.max(0, f * kellyFrac);
    b.stakeYen = Math.round(bankroll * b.stakeRatio / 100) * 100;
  });
  return ranked;
}
```

#### R-04: オッズ乖離スコア
```js
/**
 * 各艇の AI vs 市場（人気）乖離を返す。
 * positive = AI が高評価（過小評価＝妙味）
 * negative = AI が低評価（過大評価＝危険）
 */
function calcOddsDivergence(aiProbsPerBoat, oddsWin){
  if(!oddsWin) return null;
  const sumInv = Object.values(oddsWin).reduce((a,o)=>a+1/o, 0);
  const result = {};
  for(let b=1; b<=6; b++){
    const ai = aiProbsPerBoat[b-1] || 0;
    const market = oddsWin[String(b)] ? (1/oddsWin[String(b)]) / sumInv : 0;
    result[b] = {
      ai_prob: ai,
      market_prob: market,
      delta: ai - market,           // > 0 で過小評価（買い）
      ev: oddsWin[String(b)] ? ai * oddsWin[String(b)] : null,
    };
  }
  return result;
}
```

#### 6.2.2 シナリオ別 1-2-3 着分布
- シナリオ「逃げ」が起きた条件下の 1-2-3 着分布: 1着=1, 2着=2/3/4 が高頻度
- シナリオ「3 コースまくり」: 1着=3, 2着=1/4 が高頻度
- シナリオごとに別の確率テーブル → 加重平均で最終確率算出

```js
function predictWithScenarios(boats, preview, weather, sid){
  const sc = predictScenarios(...);
  const dist = {};   // {"1-2-3": prob, ...}
  Object.keys(SCENARIO_DIST).forEach(scKey => {
    const w = sc[scKey] || 0;
    Object.keys(SCENARIO_DIST[scKey]).forEach(combo => {
      dist[combo] = (dist[combo]||0) + w * SCENARIO_DIST[scKey][combo];
    });
  });
  return dist;
}
```

## 11. リスク / 緩和策

| リスク | 影響 | 緩和策 |
|------|------|--------|
| **EV モードで的中率激減** | UX 悪化 | 設定 OFF をデフォルトに、既存 UI と並列 |
| **進入予想の誤判定で全アルゴ崩壊** | 予想精度低下 | predictedConfidence を併用し低信頼時は枠ベース |
| **気象庁 API 変更** | 潮汐取得失敗 | scraper にスキーマ検証、失敗時は freshwater 扱いで継続 |
| **localStorage 容量超過** | ブラウザ強制クリア | racerDB を上位 1000 選手に制限、古いデータは IndexedDB へ移行 |
| **バックテスト計算量爆発** | UI フリーズ | Web Worker 分離、最大 180 日まで |
| **オンライン学習による発散** | 予測値が異常 | 重みクリッピング `[-10, 10]` を導入 |
| **ハードコードした場別係数の保守性** | 古くなる | nightly 統計再計算 (build-db.yml) で自動更新 |
